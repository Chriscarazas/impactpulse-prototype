# Data Visualization Rules

## Chart Principles

- Use charts to explain evidence, value, uncertainty, comparison, or action.
- Prefer direct labels and short annotations over legends.
- Always pair important charts with a text summary or data table.
- Show uncertainty beside results, not in a detached appendix.
- Never benchmark only SROI.
- Never average project SROI ratios for portfolio SROI.
- Do not use SDG colors as the dominant visual system. Analytical confidence and trade-offs matter more than icon density.

## Required Chart Semantics

- Social value: teal
- Investment or cost: blue
- Positive outcome: green
- Negative outcome or harm: red
- Evidence gap: amber
- Uncertainty: violet
- Verified state: deep green
- AI-suggested state: purple with explicit label

## SROI Charts

Headline SROI requires:

- Central value
- Plausible range
- Confidence
- Calculation boundary
- Evidence coverage
- Last approved version

Gross-to-net bridge requires:

- Gross value
- Deadweight
- Attribution
- Displacement
- Drop-off
- Net social value
- Plain-language explanation of the largest adjustment

Sensitivity charts require:

- Assumption name
- Low, central, and high values
- Effect on SROI and decision
- Evidence source and confidence

## Accessibility

- Do not rely on color alone.
- All charts need an accessible name and description.
- Use minimum 3:1 contrast for meaningful chart marks and 4.5:1 for text.
- Tables should be available for critical chart data.
- Motion must respect reduced-motion preferences.
