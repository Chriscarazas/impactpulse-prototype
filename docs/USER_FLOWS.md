# User Flows

## Flow A: Evaluative SROI

Project creation -> Location -> Evidence -> Data Quality -> Theory of Change -> Outcomes -> Valuation -> Impact Adjustments -> SROI -> Sensitivity -> Decision -> Report.

Success criteria:

- The current step is visible within five seconds.
- Evidence gaps become assigned tasks.
- Headline SROI can be traced to approved evidence, proxies, and assumptions.
- Sensitivity explains which assumptions could change the decision.

## Flow B: Forecast SROI

Project design -> Context -> Comparable evidence -> Forecast assumptions -> Scenarios -> Funding recommendation.

Success criteria:

- Forecast priors distinguish comparator evidence, external context, AI recommendation, and approved judgment.
- Weak evidence is labeled before it affects a funding recommendation.

## Flow C: Program Improvement

Monitoring signal -> Delivery issue -> Evidence interpretation -> Recommendation -> Assigned action -> Expected effect -> Review.

Success criteria:

- Recommendations include expected outcome effect, cost, equity effect, risk, confidence, owner, and monitoring indicator.
- Users can reject or challenge recommendations.

## Flow D: Assurance Reporting

Report purpose -> Readiness review -> Gap correction -> Frozen edition -> Interactive report -> Export package.

Success criteria:

- Technical readiness separates SVI principle readiness, calculation reproducibility, evidence sufficiency, and external assurance status.
- Published reports disclose methodology versions and limitations.

## Flow E: Assumption Workshop

Dashboard -> Assumptions Lab -> Live edits -> Recalculation -> Scenario comparison -> Saved draft -> Approval request.

Success criteria:

- Every assumption change shows previous value, new value, method eligibility effect, social-value effect, confidence effect, and decision effect.

## Flow F: Organization Portfolio

Organization overview -> Theme -> Country -> Project -> Evidence.

Success criteria:

- Portfolio totals disclose coverage, reporting period, excluded projects, evidence coverage, and aggregation method.
- Portfolio SROI is total adjusted present value divided by total included investment, never an average of project ratios.

## Flow G: Benchmarking

Project or portfolio -> Cohort builder -> Comparability review -> Benchmark interpretation -> Decision.

Success criteria:

- SROI comparison is blocked when boundaries, periods, outcomes, stakeholders, valuation, adjustments, costs, and evidence maturity are not aligned.
- No-match states explain why no defensible cohort exists and suggest safer comparisons.
